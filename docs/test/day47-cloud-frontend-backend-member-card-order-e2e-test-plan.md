# Day47 云端前后端订单与会员卡核销联调测试计划

测试日期：2026-07-13  
测试目标：使用当前前端代码和已部署到云服务器 Docker 的后端，创建一个用户端测试用户和一个师傅端测试用户，完整验证用户下单、师傅接单、Admin 观测与手动修改、用户端和师傅端同步变化。  
重点模块：服务订单、派单通知、会员卡购买/发放、会员卡冻结、师傅实际时长核销、后台手动调整会员卡余额。

## 1. 测试环境

后端环境：

```text
服务器：aliyun-backend
后端目录：/www/wwwroot/life-assistant
Docker 容器：life_assistant_server
公网 API：https://www.xunhaoyou.com/api
健康检查：https://www.xunhaoyou.com/api/health
```

前端环境：

```text
Admin 端：https://www.xunhaoyou.com/admin/
小程序端：当前 miniapp 代码，生产接口指向 https://www.xunhaoyou.com/api
```

如果使用本地 Admin 前端联调云端后端，需要临时把 `admin/.env.development` 的接口地址改为：

```text
VITE_APP_API_URL=https://www.xunhaoyou.com
```

如果使用服务器已部署 Admin，则直接访问：

```text
https://www.xunhaoyou.com/admin/
```

## 2. 测试账号与数据命名

所有测试数据统一使用前缀：

```text
DAY47_E2E_20260713
```

建议准备两个真实微信测试账号或两台测试手机：

| 角色 | 用途 | 命名建议 | 手机号 |
| --- | --- | --- | --- |
| 用户端测试用户 | 小程序用户下单、查看会员卡余额 | DAY47_E2E_USER | 使用实际授权手机号 |
| 师傅端测试用户 | 小程序师傅端接单、核销会员卡 | DAY47_E2E_STAFF | 使用实际授权手机号 |

1. 正式小程序默认关闭 mock 登录，用户端和师傅端应优先使用微信授权手机号登录。
2. 如果无法使用真实微信授权，需要临时开放后端 mock 登录，但这不建议在生产环境长期启用。
3. 师傅端账号需要 Admin 创建师傅资料，并保证该师傅绑定的用户手机号与测试微信授权手机号一致。

4. 选择 `DAY47_E2E_120分钟会员卡` 支付。

## 7. 保护性测试

### 7.1 师傅未填写实际时长

操作：

```text
时间类会员卡订单完成服务时，不选择或不输入实际服务时长。
```

预期：

```text
前端阻止提交，或后端返回 actualMinutes required
订单不进入待确认
会员卡余额不变化
```

### 7.2 实际核销超过冻结时长

操作：

```text
尝试提交 actualMinutes=180，而本单冻结 120 分钟。
```

预期：

```text
后端拒绝
不生成 consume/release 流水
会员卡余额不变化
```

### 7.3 Admin 扣减到小于 0

操作：

```text
当前剩余 80 分钟时，尝试扣除 -100。
```

预期：

```text
后端拒绝
页面提示余额不能为负
不生成 admin_adjust 流水
```

### 7.4 Admin 扣减到小于冻结时长

操作：

```text
另建一笔未完成会员卡订单，冻结 120 分钟后，尝试把总剩余改为 60。
```

预期：

```text
后端拒绝
页面提示总剩余不能小于已冻结
不影响已有预约订单
```

## 10. 测试数据清理

测试完成后必须清理或停用所有 `DAY47_E2E_20260713` 数据。

优先使用 Admin 页面清理：

1. 删除或取消测试订单。
2. 删除测试会员卡流水前，确认订单和会员卡状态已经不再使用。
3. 删除或停用测试用户会员卡。
4. 删除或停用测试会员卡模板。
5. 删除或停用测试服务。
6. 删除或停用测试师傅。
7. 删除或停用测试用户。

如果 Admin 页面不支持完全删除，则保留数据但必须改为停用，并在备注中标记：

```text
DAY47_E2E_20260713 测试数据，已停用
```

禁止手动删除非 `DAY47_E2E_20260713` 前缀的数据。

## 11. 最终验收标准

本次测试通过需要同时满足：

1. 用户端能登录、看到会员卡、使用会员卡下单。
2. 订单创建后会员卡正确冻结。
3. Admin 能看到订单、派单、会员卡冻结和流水。
4. 师傅端能收到派单，完成接单、出发、开始、上传照片、填写实际时长、完成服务。
5. 系统按实际服务时长核销会员卡，并释放未用冻结时长。
6. 用户端订单详情和卡包显示核销后的正确余额。
7. 师傅端任务详情显示核销后的正确余额。
8. Admin 手动增加、扣除、指定余额后，用户端和师傅端刷新后同步显示新余额。
9. 会员卡流水完整记录 `grant/freeze/consume/release/admin_adjust`。
10. 保护性测试能阻止超额核销、负余额扣减、低于冻结余额调整。
11. 测试完成后，测试数据已清理或明确停用。
