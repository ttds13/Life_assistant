# Day44 线下客户与订单录入简化执行测试报告

测试时间：2026-07-09  
执行范围：Admin 用户管理、Admin 服务预约订单、后端会员卡查询、线下订单闭环 smoke  
测试结论：通过  
测试数据状态：本次 smoke 数据已清理

## 1. 本次实现内容

### 1.1 客户快速录入

文件：`admin/src/views/life/resource/index.vue`

已完成：

1. 新增用户入口改为“快速新增线下客户”。
2. 常用录入字段压缩为客户姓名、手机号、服务地址、门牌号、客户备注。
3. 来源、城市编码、联系人、联系电话、城市、区县、小区/地点放入“高级选项”。
4. 默认来源改为 `offline`。
5. 昵称为空时用手机号。
6. 联系人、联系电话、地址标题随手机号、昵称、地址自动补齐。
7. 地址为空时允许只创建客户。
8. 门牌号已填写但服务地址为空时阻止提交。
9. 手机号输入后自动查询已有客户，命中时提示“保存时将复用该客户”。
10. 后端已有手机号时继续复用客户，并可追加本次填写的新服务地址。

### 1.2 订单可视化录入

文件：`admin/src/views/life/orders/index.vue`

已完成：

1. “录入外来订单”改为 4 个分区：选择客户、选择服务、地址和预约时间、收款方式。
2. 常规界面不再让管理员手填用户 ID、服务 ID、地址 ID、用户卡 ID。
3. 客户通过远程搜索选择，支持手机号和姓名搜索。
4. 找不到客户时，可直接填写手机号、姓名和地址，由提交时创建或复用客户。
5. 服务通过远程搜索选择，仅拉取上架服务。
6. 选择服务后自动带出服务价格，并按服务时长计算预约结束时间。
7. 选择客户后自动加载客户地址，默认选择默认地址或第一条地址。
8. 支持填写新增服务地址；填写新增地址时提交地址对象，未填写新增地址时提交已选地址 ID。
9. 支持线下已收款、待线下收款、会员卡抵扣 3 种支付方式。
10. 线下已收款默认收款备注为“线下录入已收款”。
11. 会员卡抵扣只加载当前客户名下、与当前服务计卡类型匹配的可用会员卡。
12. 应收金额默认等于服务价格；如改价，必须填写后台备注说明原因。
13. 提交前展示客户、服务、地址、预约、支付方式、创建后状态摘要。

### 1.3 后端归属过滤

文件：`server/src/admin-business/admin-business.service.ts`

已完成：

1. `listUserMemberCards(query)` 增加 `query.userId` 过滤。
2. Admin 订单录入页请求用户会员卡时，后端只返回当前用户名下会员卡。
3. 前端仍保留二次过滤，后端仍在下单冻结会员卡时做最终业务校验。
