## 4. 首页目标结构

后续实现时建议将首页改为以下顺序：

```txt
page
  custom-nav-placeholder / safe-area

  sticky-top
    location-search-header
      address-row
      search-row
      cart-entry

  scroll-content
    hero-banner
    guarantee-strip
    service-category-card
    housekeeping-card
    operations-grid
      nearby-staff-card
      direct-service-card
      seasonal-promotion-card
    coupon-pass-card
    hot-service-list
```

其中 `sticky-top` 是本次首页改造的关键。无论用户在首页首屏还是向下滚动到 `home2`、`home3` 的位置，都应该能看到地址与搜索模块。

推荐实现方式：

```txt
顶部模块使用 position: sticky
top: 0
z-index: 20
背景保持 #FF3B3F / #FF373D 红色
下方内容使用 #F5F5F5 或 #F5F7FA 浅灰背景
```

如果微信小程序端对 `position: sticky` 表现不稳定，备选方案为：

```txt
顶部模块放在页面固定顶部
内容区域增加 padding-top
滚动区域只滚动顶部模块下方内容
```

```txt

数据：getServices({ page: 1, pageSize: 10 })
```

新位置：

```txt

作为真实服务承接区
```

处理规则：

```txt
接口正常：展示热门服务列表
接口为空：隐藏列表或展示 empty-state
接口失败：首页主体运营模块仍然展示，只在热门服务区域提示加载失败
```

这样可以避免接口异常导致整个首页首屏为空。
