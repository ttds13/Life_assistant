# 线下客户与外来订单录入功能闭环说明

更新时间：2026-07-09  
适用对象：后台管理员、客服、门店人员、运营负责人  
适用模块：用户管理、订单管理、服务管理、营销管理、财务管理、师傅端通知

## 1. 功能定位

本次新增功能主要解决线下业务录入繁琐、容易填错的问题。

过去后台录入订单时，需要管理员手动填写用户 ID、服务 ID、地址 ID、会员卡 ID。普通客服或门店人员不容易理解这些内部编号，数据量增加后容易出现填错客户、填错服务、填错会员卡的问题。

现在系统改为可视化录入：

```text
选择客户 -> 选择服务 -> 选择地址和预约时间 -> 选择收款方式 -> 创建订单 -> 派师傅 -> 履约 -> 财务记账
```

管理员不需要记内部 ID，只需要按业务顺序选择客户、服务、地址和支付方式。

## 2. 适用业务场景

### 2.1 小程序线上订单

客户自己在微信小程序注册、购买、预约服务。

后台主要负责：

1. 查看订单。
2. 派师傅。
3. 跟进履约。
4. 处理退款、售后、财务统计。

### 2.2 线下手动录入订单

客户没有从小程序下单，而是通过电话、门店、微信私域、转账、老客户复购等方式成交。

后台人员可以直接在后台完成：

1. 新增或复用客户。
2. 选择服务项目。
3. 录入服务地址。
4. 选择收款方式。
5. 创建订单。
6. 派师傅。
7. 形成完整财务、积分、会员卡、通知和履约记录。

## 3. 完整业务闭环

### 3.1 线下已收款闭环

适用场景：客户已经通过现金、门店、转账、微信私域等方式付款。

闭环如下：

```text
后台录入客户
-> 创建线下订单
-> 选择“线下已收款”
-> 系统生成支付记录
-> 订单进入待派单
-> 后台派师傅或自动派单
-> 师傅端收到派单通知
-> 师傅接单、出发、服务、完成
-> 用户确认完成
-> 系统生成积分、师傅收入、财务统计
```

结果：

1. 订单可在订单列表和订单详情查看。
2. 收款可在支付记录、财务统计中查看。
3. 派单可在派单记录和师傅通知中查看。
4. 服务过程可在履约记录中查看。
5. 完成后会进入财务、积分、师傅收入统计。

### 3.2 待线下收款闭环

适用场景：客户先预约，还没有付款。

闭环如下：

```text
后台创建订单
-> 选择“待线下收款”
-> 订单进入待支付
-> 未收款前不能派单
-> 收款后在订单详情确认线下收款
-> 订单进入待派单
-> 后续进入派单、履约、财务闭环
```

注意：

1. 待收款订单不会直接派师傅。
2. 必须先在订单详情中确认收款。
3. 确认收款后才会进入待派单状态。

### 3.3 会员卡抵扣闭环

适用场景：客户已经购买或持有会员卡，本次服务使用会员卡权益抵扣。

闭环如下：

```text
选择已有客户
-> 选择支持会员卡的服务
-> 选择“会员卡抵扣”
-> 系统只展示当前客户可用会员卡
-> 创建订单后冻结会员卡额度
-> 服务完成后核销会员卡额度
-> 如订单取消或退款，系统按规则释放或回滚
```

注意：

1. 会员卡不能手填编号。
2. 只能选择当前客户名下的可用会员卡。
3. 服务是否支持会员卡，由服务项目和会员卡规则共同决定。
4. 不建议人工直接修改会员卡余额，应通过订单流程完成抵扣。
5. 会员卡抵扣是“用卡做服务”，不是现金收款，不要再点击“确认线下收款”。

### 3.4 线下购买会员卡闭环

客户在线下购买会员卡时，从“会员卡购买订单”录入，不从“服务预约订单”录入。

```text
会员卡购买订单 -> 录入会员卡购买 -> 选客户 -> 选会员卡 -> 选收款状态 -> 创建订单
```

线下已收款会自动生成支付流水并发卡；待线下收款需要收款后在详情页确认，再发卡。

## 4. 后台操作说明

### 4.1 新增线下客户

入口：

```text
用户管理 -> 用户列表 -> 新增客户
```

常规只需要填写：

| 字段 | 说明 |
| --- | --- |
| 手机号 | 必填，是识别客户的核心信息 |
| 客户姓名 | 可选，不填时系统会使用手机号 |
| 服务地址 | 可选，建议填写，方便后续下单 |
| 门牌号 | 可选，用于上门定位 |
| 客户备注 | 可选，用于记录来源、销售、特殊说明 |

系统会自动处理：

1. 默认客户来源为线下客户。
2. 自动生成联系人和联系电话。
3. 自动生成地址标题。
4. 手机号已存在时，系统会提示并复用原客户，不会重复创建客户。

### 4.2 录入外来订单

入口：

```text
订单管理 -> 服务预约订单 -> 录入订单
```

按页面顺序操作：

1. 搜索并选择客户。
2. 搜索并选择服务项目。
3. 选择客户已有地址，或填写新增服务地址。
4. 选择预约开始时间，系统会按服务时长自动计算结束时间。
5. 选择收款方式。
6. 检查提交摘要。
7. 点击创建订单。

提交前摘要会展示：

1. 客户姓名和手机号。
2. 服务名称和价格。
3. 服务地址。
4. 预约时间。
5. 支付方式。
6. 创建后的订单状态。

### 4.3 创建后处理

订单创建后，根据支付方式进入不同状态：

| 支付方式 | 创建后状态 | 后续操作 |
| --- | --- | --- |
| 线下已收款 | 待派单 | 可以派师傅 |
| 待线下收款 | 待支付 | 先确认线下收款，再派师傅 |
| 会员卡抵扣 | 待派单 | 可以派师傅，会员卡额度会被冻结 |

### 4.4 录入线下会员卡购买

入口：

```text
订单管理 -> 会员卡购买订单 -> 录入会员卡购买
```

简要操作：

1. 选择已有客户。
2. 选择会员卡。
3. 选择“线下已收款”或“待线下收款”。
4. 确认金额后创建订单。

注意：

1. 买会员卡走“会员卡购买订单”。
2. 用会员卡预约服务走“服务预约订单”的“会员卡抵扣”。
3. 两个入口不要混用。

## 5. 使用注意事项

### 5.1 客户手机号必须准确

手机号是线下客户复用和去重的主要依据。

注意：

1. 新建客户前优先搜索手机号。
2. 同一个客户不要使用多个手机号重复建档。
3. 如果手机号输错，后续订单、会员卡、积分、财务记录都会挂到错误客户名下。

### 5.2 服务项目必须从列表选择

订单金额、服务时长、会员卡规则都来自服务项目。

注意：

1. 不要随意创建临时服务代替正式服务。
2. 服务价格或时长变更前，应确认是否会影响后续订单录入。
3. 已经产生订单的服务不建议删除，建议停用。

### 5.3 地址要尽量完整

地址用于师傅上门服务。

建议填写：

1. 小区或地点。
2. 详细地址。
3. 楼栋、单元、门牌号。
4. 特殊入口或停车说明可写在备注中。

### 5.4 金额改动必须写明原因

系统默认使用服务价格作为应收金额。

如果实际收款金额和服务价格不一致，必须在后台备注写明原因，例如：

1. 老客户优惠。
2. 门店活动价。
3. 补差价。
4. 多项目合并收款。

这样后续财务对账时可以追溯。

### 5.5 待收款订单不能提前派单

为了避免未收款就派师傅，系统会拦截待支付订单派单。

正确流程：

```text
待线下收款订单 -> 收到款 -> 订单详情确认线下收款 -> 再派师傅
```

### 5.6 会员卡不能跨客户使用

系统只允许选择当前客户名下的会员卡。

注意：

1. 不要把 A 客户的卡用于 B 客户订单。
2. 如果客户使用家属卡或公司卡，需要先在业务上确认归属，再由管理员按规则处理。
3. 不要直接修改会员卡余额代替订单抵扣。

### 5.7 不要绕开订单流程记账

线下收款、会员卡抵扣、积分发放、师傅收入都依赖订单闭环自动生成。

不要用以下方式替代订单流程：

1. 只在备注里写已收款，但不确认收款。
2. 手工改订单状态跳过支付。
3. 手工扣会员卡余额。
4. 手工改财务统计。

否则会导致订单、支付、会员卡、积分和财务统计不一致。

## 6. 管理检查建议

运营负责人建议定期检查以下内容：

| 检查项 | 入口 | 目的 |
| --- | --- | --- |
| 线下订单 | 订单管理 / 服务预约订单 | 查看线下录入是否规范 |
| 待支付订单 | 订单管理 / 服务预约订单 | 检查是否有已收款但未确认的订单 |
| 待派单订单 | 订单管理 / 待派单 | 确认收款后是否及时派师傅 |
| 支付记录 | 财务管理 / 支付记录 | 核对线下收款流水 |
| 财务统计 | 财务管理 / 财务统计 | 查看线下收入是否计入统计 |
| 用户会员卡 | 营销管理 / 用户会员卡 | 核对会员卡冻结、核销、释放情况 |
| 师傅通知 | 师傅管理 / 师傅通知 | 确认派单通知是否发送和已读 |

## 7. 本功能带来的改进

本次改造后，线下业务可以形成完整闭环：

1. 客服录入更简单，不需要理解内部 ID。
2. 客户、服务、地址、会员卡都通过列表选择，减少录错。
3. 线下已收款、待收款、会员卡抵扣三种场景都能进入统一订单流程。
4. 派单通知、师傅履约、支付记录、积分、会员卡流水、师傅收入、财务统计可以自动串联。
5. 后续运营和财务可以根据后台记录追溯每一笔线下订单。

最终目标是让线下成交不再停留在纸面、微信聊天或人工表格中，而是统一进入后台系统，形成可查询、可派单、可履约、可对账的业务闭环。
