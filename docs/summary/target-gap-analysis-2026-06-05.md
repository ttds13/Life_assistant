# Target 标准与当前项目实现差距评估

更新日期：2026-06-05  
对照标准：`docs/standards/target.md`  
评估范围：`server`、`miniapp`、`admin` 当前代码实现

## 1. 总体结论

当前项目已经具备“本地演示级业务闭环”的主要骨架，尤其是订单、支付 mock、自动派单、手动接单、师傅履约、admin 订单管理和 admin 基础资源管理已经明显推进。

但如果按 `target.md` 的完整标准衡量，当前还不是完整的生活服务平台。主要差距不在“有没有页面”，而在“是否真实接入、是否形成用户-师傅-admin-支付-售后-运营的生产级闭环”。

当前状态可以概括为：

```txt
用户端基础下单链路：较完整，但营销、评价、售后、收藏、消息、真实支付仍缺
师傅端接单履约链路：可演示，但生产级员工账号、日程、收入、导航仍缺
管理后台：订单和资源管理推进较多，但导出、运营深度、投诉工单闭环仍不足
后端数据模型：覆盖面较广，但不少模型还没有前端真实闭环
生产化能力：仍缺真实微信支付、HTTPS 部署、消息订阅、正式 staff auth
```

## 3. 与 target 的主要差距

### 3.1 用户端注册登录

target 标准：

```txt
支持手机号注册/登录，以及微信一键授权登录。
```

当前实现：

```txt
微信 code2session + 获取手机号接口已实现
开发模拟登录已实现
token、用户 store、401 处理已实现
```

差距：

```txt
真实微信手机号授权仍依赖微信平台能力和真机环境完整验收
生产环境登录配置、合法域名、隐私协议和提审配置仍未完成
没有普通短信验证码登录
mock-login 仍在开发链路中存在，生产需确保关闭
```

优先级：P0。

### 3.2 首页、服务分类与搜索

target 标准：

```txt
首页展示服务分类、热门服务、优惠活动和个人化推荐。
服务分类详细导航，支持关键词搜索。
```

当前实现：

```txt
服务分类和服务列表已接真实接口
主页关键词搜索已跳转到搜索结果页
搜索结果页展示服务卡片
基础关键词匹配已实现
```

差距：

```txt
热门服务、优惠活动、个人化推荐仍偏静态或提示建设中
搜索算法只是本地基础关键词匹配，没有搜索日志、热词、排序权重
首页多个活动入口仍是 showToast 建设中
服务分类筛选和服务搜索之间还没有完整的统一检索体验
```

优先级：P1。

## 7. 建议补齐顺序

### P0：先补真实闭环

```txt
1. 接入真实微信支付：预支付单、前端调起、回调验签、幂等、异常单
2. 完成 staff 生产级登录：员工账号密码登录、staff token、staff guard
3. 对用户支付成功 -> 自动派单 -> 师傅接单 -> 履约 -> 用户确认做完整真机验收
4. 下线生产环境 mock-login、mock-pay、dev-staff-session
5. 部署 HTTPS 后端和小程序合法域名
```

### P1：补运营闭环

```txt
1. 优惠券领取、使用、抵扣、过期、核销
2. 会员卡购买、次数扣减、消费记录
3. 用户评价提交和图片上传
4. 用户售后工单提交、admin 处理、师傅回应
5. 服务详情补齐服务范围、须知、评价摘要、师傅信息
6. 师傅收入明细、提现申请、评价查看、导航入口真实化
```

### P2：补平台增强

```txt
1. 消息通知和小程序订阅消息
2. 收藏服务/收藏师傅
3. 智能估价和价格规则管理
4. 活动管理和个性化推荐
5. 员工绩效分析和经营数据导出
6. 调度算法升级：距离、时间、技能、负载、拒单历史
```

## 8. 当前最短可试运营路径
