## 2. 两个业务渠道

系统现在按两个渠道管理业务。

| 渠道 | 入口 | 适用场景 | 后台重点 |
| --- | --- | --- | --- |
| 小程序自助渠道 | 客户自己在微信小程序注册、购买、预约 | 正常线上下单、线上支付、会员卡抵扣 | 后台查看订单、派师傅、处理退款售后、做财务统计 |
| 线下手动录入渠道 | 客服或门店在后台录入 | 电话预约、门店收款、微信私域转账、老客户线下购买 | 后台新建客户、录入订单、确认线下收款、派师傅、形成财务记录 |

两个渠道最终都必须进入同一套闭环：

```text
客户信息 -> 服务/会员卡 -> 订单 -> 支付或会员卡抵扣 -> 派师傅 -> 履约 -> 财务/积分/售后记录
```

## 3. 客户管理

### 3.1 查询客户

入口：用户管理 / 用户列表

操作步骤：

1. 优先用手机号搜索客户。
2. 如果客户已经存在，后续订单、会员卡、地址都使用这个客户。
3. 不要为同一个手机号重复创建多个客户。

### 3.2 新增线下客户

入口：用户管理 / 用户列表 / 新增客户

建议填写：

| 字段 | 要求 |
| --- | --- |
| 昵称或姓名 | 可以填客户姓名；不填时系统可使用手机号 |
| 手机号 | 必填，作为线下客户的主要识别信息 |
| 来源 | 选择线下、电话、微信私域、门店等真实来源 |
| 城市信息 | 按客户服务城市填写 |
| 后台备注 | 记录客户来源、销售人员、线下成交说明 |
| 创建地址 | 如客户已经确定服务地址，建议同时创建 |

### 3.3 地址管理

入口：用户管理 / 地址管理，或用户列表中进入客户地址。

地址建议包含：

1. 联系人姓名。
2. 联系电话。
3. 城市、区县。
4. 小区或地址标题。
5. 详细地址、门牌号。

地址定位不准确时，以人工填写的详细地址为准，并在后台备注中补充楼栋、门牌、特殊入口等信息。

## 4. 服务管理

入口：服务管理 / 服务分类、服务项目

日常操作：

1. 先维护服务分类，再维护服务项目。
2. 服务项目需要设置名称、价格、服务时长、上下架状态。
3. 如果服务支持会员卡抵扣，需要在会员卡模板中配置对应服务规则。
4. 已经产生订单的服务，不建议随意删除；需要停用时优先下架。

注意事项：

1. 服务项目是订单和会员卡规则的基础数据。
2. 价格或会员卡抵扣规则调整后，只影响新订单；历史订单以订单详情中的快照和流水为准。

## 5. 小程序自助订单流程

适用场景：客户在小程序中自行预约服务或购买会员卡。

标准流程：

```text
客户小程序下单 -> 后台订单列表出现订单 -> 后台派师傅 -> 师傅端接收通知 -> 师傅履约 -> 订单完成 -> 财务统计更新
```

后台操作：

1. 进入订单管理 / 服务预约订单。
2. 查看订单来源、客户、服务、预约时间、地址、支付状态。
3. 如果订单状态为待派单，进入订单详情或待派单列表。
4. 选择合适师傅，点击派单。
5. 派单后可在订单详情查看师傅通知发送状态、阅读状态、重发次数。
6. 服务完成后，在订单详情查看履约时间线、照片、支付、积分、退款、会员卡流水。

## 6. 线下手动录入订单流程

适用场景：客户没有从小程序下单，由客服、门店或销售人员在后台代录入。

标准流程：

```text
查询或新建客户 -> 维护客户地址 -> 创建服务预约订单 -> 选择线下支付方式 -> 确认收款或待收款 -> 派师傅 -> 履约 -> 财务统计
```

操作步骤：

1. 在用户管理 / 用户列表中用手机号查询客户。
2. 如果不存在，先新增客户并维护地址。
3. 进入订单管理 / 服务预约订单，创建服务预约订单。
4. 选择客户、服务项目、预约时间、服务地址。
5. 选择订单来源，例如线下、电话、门店、微信私域。
6. 按真实收款情况选择支付方式。

线下订单支付方式说明：

| 方式 | 使用场景 | 后续动作 |
| --- | --- | --- |
| 线下已收款 | 客户已经现金、转账或门店付款 | 系统生成支付记录，订单可进入待派单 |
| 待线下收款 | 客户已预约但尚未付款 | 订单先保持待支付，收款后在订单详情确认线下收款 |
| 会员卡抵扣 | 客户已有可用会员卡 | 选择用户会员卡，订单会记录会员卡冻结、扣减或释放流水 |

重要要求：

1. 线下已收款必须录入真实金额和备注。
2. 不要直接绕过支付记录修改订单金额。
3. 会员卡抵扣必须通过订单流程使用用户会员卡，不要手工减少卡余额。
4. 派单后，师傅端应收到通知；后台可在师傅通知页面查看和重发。

## 7. 派单与师傅通知

入口：

1. 订单管理 / 待派单。
2. 订单管理 / 服务预约订单 / 详情。
3. 师傅管理 / 师傅通知。

派单步骤：

1. 打开待派单订单。
2. 查看预约时间、地址、服务项目和客户备注。
3. 选择可服务的师傅。
4. 点击确认派单。
5. 派单后检查订单详情中的师傅、派单记录、通知状态。

通知检查：

1. 进入师傅管理 / 师傅通知。
2. 可按师傅 ID、订单 ID、通知状态查询。
3. 如果派单通知失败或师傅未收到，可点击重发派单通知。
4. 通知详情中可查看发送状态、是否已读、发送时间、阅读时间、失败原因和重发次数。

## 8. 师傅资料复核

入口：师傅管理 / 资料变更复核

适用场景：

1. 师傅修改头像。
2. 师傅修改姓名。
3. 师傅修改工作状态或资料。
4. 认证资料通过后，后续资料变更需要后台复核。

处理方式：

1. 查看变更前和变更后的资料快照。
2. 确认资料真实、合规后通过。
3. 资料不完整或不合规时驳回，并填写驳回原因。
4. 复核记录会保留，方便后续追溯。

## 9. 会员卡管理

入口：

1. 营销管理 / 会员卡模板。
2. 营销管理 / 用户会员卡。
3. 营销管理 / 会员卡流水。
4. 订单管理 / 会员卡购买订单。

模块关系：

| 模块 | 作用 |
| --- | --- |
| 会员卡模板 | 定义可售卖的卡、价格、单位、总权益、适用服务规则 |
| 用户会员卡 | 某个客户实际拥有的卡，记录余额、冻结额度、有效期 |
| 会员卡流水 | 记录发卡、冻结、扣减、释放、退款等变动 |
| 服务预约订单 | 使用会员卡抵扣时引用用户会员卡并生成流水 |
| 会员卡购买订单 | 客户购买会员卡时产生的订单和支付记录 |

日常操作：

1. 新卡种先在会员卡模板中维护。
2. 线下售卡时，在用户会员卡中给客户发放会员卡，并记录来源和线下收款信息。
3. 客户预约服务使用会员卡时，应通过订单选择用户会员卡。
4. 服务完成后，系统按订单实际履约情况扣减会员卡。
5. 取消或退款时，检查会员卡流水是否有释放或回滚记录。

注意事项：

1. 不要把会员卡模板当成客户卡使用。
2. 不要直接修改用户会员卡余额来代替订单扣减。
3. 会员卡与服务项目的适用规则要提前维护，否则下单时可能无法正确抵扣。

## 10. 优惠券与积分

优惠券入口：

1. 营销管理 / 优惠券。
2. 营销管理 / 用户券明细。

优惠券操作：

1. 在优惠券中维护券名称、优惠金额、使用门槛、有效期、状态。
2. 在用户券明细中查看客户领券、用券、过期、核销情况。
3. 小程序订单使用优惠券后，订单详情和财务统计会记录优惠金额。

积分入口：

1. 用户管理 / 用户列表 / 用户积分。
2. 财务管理 / 积分流水。

积分操作：

1. 查看客户当前积分、累计金额和最近流水。
2. 必要时可做后台积分调整，并填写调整原因。
3. 订单完成、退款、人工调整都会进入积分流水。

注意事项：

1. 积分调整必须写清楚原因。
2. 退款或取消订单后，需要检查积分是否扣回或产生回收流水。
3. 优惠券抵扣和积分变动最终都应能在订单详情和财务统计中查到。

## 11. 财务管理

入口：

1. 财务管理 / 财务统计。
2. 财务管理 / 支付记录。
3. 财务管理 / 退款审核。
4. 财务管理 / 提现管理。
5. 财务管理 / 积分流水。

财务统计可查看：

1. 订单原价。
2. 优惠金额。
3. 历史实收。
4. 退款金额。
5. 净收入。
6. 师傅收入。
7. 积分净值。
8. 用券订单。
9. 按订单来源、支付渠道、订单状态、退款状态、提现状态拆分的数据。

日常核对建议：

1. 每日核对支付记录和线下收款记录。
2. 每日核对退款审核是否处理完。
3. 每日核对提现申请是否完成审核和打款。
4. 每周查看订单来源统计，区分小程序订单和线下录入订单。

退款处理：

1. 进入财务管理 / 退款审核。
2. 查看退款订单、退款金额、原因和支付记录。
3. 审核通过或驳回。
4. 退款完成后，检查订单详情、支付记录、积分流水、会员卡流水是否同步更新。

提现处理：

1. 进入财务管理 / 提现管理。
2. 查看师傅提现单和关联收入记录。
3. 审核提现。
4. 执行打款或人工处理。
5. 失败、超时或客户确认中的提现，需要按状态重试或人工核销。

## 13. 每日运营检查清单

建议每天至少检查一次：

1. 订单管理 / 待派单：是否有未派单订单。
2. 师傅管理 / 师傅通知：是否有发送失败或长期未读通知。
3. 财务管理 / 退款审核：是否有待审核退款。
4. 财务管理 / 提现管理：是否有待审核或失败提现。
5. 营销管理 / 用户会员卡：是否有异常冻结额度。
6. 财务管理 / 财务统计：当天订单、实收、退款、净收入是否匹配。
7. 评价与售后 / 售后工单：是否有未处理工单。

## 14. 禁止绕过的操作

为保证财务和业务闭环，不建议执行以下操作：

1. 不要为同一个手机号重复创建客户。
2. 不要在已支付订单中直接改金额来代替退款或补款。
3. 不要手工修改会员卡余额来代替订单核销。
4. 不要线下收款后不录入支付记录。
5. 不要派单后只电话通知师傅而不在系统内派单。
6. 不要删除已经产生支付、退款、会员卡流水或师傅收入的正式订单。
7. 不要把测试数据留在正式生产环境。

## 15. 常见问题处理

### 15.1 登录失败

1. 确认访问地址是 https://www.xunhaoyou.com/admin/。
2. 确认账号是 `admin`。
3. 确认密码是 `Admin123456789`。
4. 如果仍失败，联系技术人员检查后台服务和账号状态。

### 15.2 客户找不到

1. 优先用手机号搜索。
2. 如果客户来自小程序，检查是否使用了微信绑定手机号。
3. 如果客户来自线下，确认是否已经由后台创建过客户。
4. 确认没有使用不同手机号重复录入。

### 15.3 派单后师傅说没收到

1. 打开订单详情，查看师傅通知状态。
2. 进入师傅管理 / 师傅通知，按订单 ID 查询。
3. 如果通知失败或未读，可点击重发派单通知。
4. 仍无法收到时，联系技术人员检查师傅端登录状态和通知服务。

### 15.4 财务统计金额不一致

1. 先按日期筛选财务统计。
2. 检查支付记录是否包含全部线上和线下收款。
3. 检查退款记录是否已完成。
4. 检查订单来源是否选错。
5. 检查是否存在人工改金额、未确认线下收款、未走订单流程的会员卡扣减。
