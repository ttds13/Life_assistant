# Day 22 - 小程序首页改造计划：参考 home.jpg

## 1. 背景

首页需要改回更接近 `photo/home.jpg` 的家政服务型布局，而不是简单的“顶部 + Banner + 全部类目平铺”。

新的方向是：

- 顶部固定地址 + 搜索框 + 购物车。
- 顶部下面是一张大海报 Banner。
- Banner 下方是服务模块。
- 服务模块分为“热门服务”和“更多服务”。
- 热门服务使用较大的服务卡片。
- 更多服务使用较小的服务入口卡片。

本计划只更新首页结构和服务入口组织方式，不改支付、订单、登录、后台服务管理。

## 2. 视觉参考

参考图：

```text
photo/home.jpg
```

参考图中的关键特征：

- 顶部红色区域承载地址、搜索、购物车。
- 地址显示在搜索框上方。
- 搜索框是白色圆角输入样式。
- 购物车入口在搜索框右侧。
- Banner 是首屏主体视觉，突出一个服务和价格/文案。
- Banner 下方有服务保障标签。
- 服务区域使用卡片化模块。
- 底部保留 tabbar。

## 4. 模块说明

### 4.1 顶部固定区

顶部固定区需要接近 `home.jpg`：

- 第一行：地址 / 定位。
- 第二行：搜索框 + 购物车。
- 搜索框在左侧，占主要宽度。
- 购物车在右侧，使用图标入口。
- 顶部区域可以固定在页面顶部，也可以随页面自然滚动；第一版优先保持简单，如果固定导致兼容问题则不强制 sticky。

交互：

- 点击地址：暂时可保留占位，不强制接真实定位。
- 点击搜索框：进入服务搜索页。
- 点击购物车：如果购物车功能未完成，先 toast 提示“购物车功能待完善”；不要跳空页面。

颜色：

- 参考 `home.jpg` 的红色顶部区域。
- 但要结合当前项目已有主色，避免引入过多新颜色。
- 如果当前项目主色仍是蓝色，需要在实现前确认是否允许首页顶部改红。

## 4.2 大海报 Banner

Banner 放在顶部固定区下方。

内容：

- 主服务标题，例如：日常保洁服务。
- 价格或引导文案，例如：50 元 / 小时，3 小时起约。
- 主按钮，例如：立即预约 / 立即抢购。
- 右侧放服务图标或服务场景图。

交互：

- 点击 Banner 主按钮进入对应服务详情或服务列表。
- 如果暂时没有指定服务，可以跳到热门服务列表的第一个服务。

设计：

- Banner 是首页第一视觉重点。
- 不要做多个轮播，先做单张海报。
- 可使用已有服务封面、默认图或简洁图标。

## 4.3 服务保障条

参考 `home.jpg`，Banner 下方可保留一行轻量保障信息：

```text
不满意重做 / 过期退 / 随时退 / 全场保障
```

第一版可以静态展示。

如果空间不够，也可以先不做，但计划中保留这个模块。

## 4.4 热门服务

热门服务是用户最容易点击的入口。

布局：

- 标题：热门服务。
- 2 列大卡片。
- 每张卡片展示：
  - 服务图片或图标。
  - 服务名称。
  - 价格：`¥xx起`。
  - 简短描述可选。

数据来源：

```ts
getServices({ page: 1, pageSize: 4 })
```

如果后端后续支持排序字段，可按 `sortOrder` 或推荐标识取前 4 个。

交互：

```text
点击热门服务卡片 -> /pages/service/detail?id={service.id}
```

## 4.5 更多服务

更多服务用于承载全部服务类目或更多服务入口。

布局：

- 标题：更多服务。
- 3 列或 4 列小卡片。
- 小卡片展示：
  - 图标。
  - 名称。

数据来源优先级：

1. 如果展示服务类目：使用 `getServiceCategories()`。
2. 如果展示服务项：使用 `getServices({ page, pageSize })`。

本次建议：更多服务先展示服务类目。

交互：

```text
点击类目小卡 -> /pages/service/search?categoryId={category.id}
```

## 5. 数据设计

首页需要同时请求：

```ts
getServiceCategories()
getServices({ page: 1, pageSize: 4 })
```

页面状态：

```ts
categories: ServiceCategory[]
hotServices: Service[]
loading: boolean
isError: boolean
```

空状态：

- 如果类目和热门服务都为空，显示“暂无服务”。
- 如果热门服务为空但类目存在，只展示更多服务。
- 如果类目为空但热门服务存在，只展示热门服务。

## 7. 实施步骤

### Step 1 - 恢复顶部 home.jpg 结构

- 顶部显示地址。
- 搜索框放在地址下方。
- 搜索框右侧增加购物车图标。
- 搜索框点击进入搜索页。
- 购物车点击先给待完善提示。

### Step 2 - 添加首页 Banner

- 添加单张大海报。
- 放在顶部区域下方。
- 使用一个主服务卖点。
- 主按钮进入服务详情或服务列表。

### Step 3 - 添加保障条

- 在 Banner 下方添加轻量保障文案。
- 使用图标 + 文案。

### Step 4 - 重做热门服务区

- 拉取前 4 个服务。
- 以 2 列大卡片展示。
- 点击进入服务详情。

### Step 5 - 重做更多服务区

- 拉取全部服务类目。
- 以小卡片展示。
- 点击进入类目服务列表页。

### Step 6 - 服务列表页兜底

- 确认 `/pages/service/search` 存在。
- 支持 `categoryId` 查询。
- 点击具体服务进入详情。

### Step 7 - 真机检查

重点检查：

- 顶部地址、搜索框、购物车是否和 `home.jpg` 的层级一致。
- Banner 是否在首屏有足够视觉权重。
- 热门服务大卡片是否适合点击。
- 更多服务是否能完整下滑。
- 文字是否不重叠、不溢出。

## 8. 验收标准

完成后应满足：

- 顶部有地址/定位。
- 顶部有搜索框。
- 搜索框右侧有购物车入口。
- Banner 位于顶部区域下方。
- Banner 有明确家政/清洁/维修服务主视觉。
- 热门服务区域使用大服务卡片。
- 更多服务区域使用小服务卡片。
- 点击热门服务进入服务详情。
- 点击更多服务类目进入服务列表。
- 搜索入口可进入服务搜索页。
- 购物车未完成时有明确提示，不跳空页面。
- `pnpm type-check` 通过。
- 微信开发者工具和真机预览布局正常。

## 9. 暂不处理

本次暂不处理：

- 真实定位和地址选择。
- 购物车完整业务。
- Banner 后台配置化。
- 首页复杂轮播。
- 首页优惠券和钱包入口。
- 推荐算法。
- 支付流程。
- 订单流程。

## 10. 风险与注意事项

- 不要再把首页改成只有类目平铺的极简页。
- 不要把类目 ID 当服务 ID 跳服务详情。
- 不要做过多营销模块，首版只保留 Banner、热门服务、更多服务。
- 如果引入红色顶部，需要确认和当前品牌主色是否冲突。
- 如果真机首屏拥挤，优先压缩 Banner 高度，不压缩顶部搜索区。
